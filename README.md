# sertas

Кроссплатформенное десктоп-приложение для голосовых и видео-звонков с
демонстрацией экрана — аналог Discord. До 4 участников в P2P-меше на WebRTC.
Цель — качественная демонстрация экрана с высоким FPS и чистым системным звуком.

Платформы: macOS и Windows (Linux — позже).

Документы: [дизайн](docs/superpowers/specs/2026-06-24-discord-analog-voice-app-design.md),
[план Фазы 0](docs/superpowers/plans/2026-06-24-phase-0-signaling-and-foundations.md),
[деплой сервера](DEPLOY.md).

## Стек

JDK 21, JavaFX 21, [webrtc-java](https://github.com/devopvoid/webrtc-java) (нативный
libwebrtc), Javalin (сигналинг-сервер), Gradle (wrapper в комплекте — ставить
Gradle не нужно).

## Модули

| Модуль | Ответственность |
|--------|-----------------|
| `protocol` | Сообщения сигналинга + JSON-кодек |
| `signaling-server` | WebSocket-сервер: комнаты, relay SDP/ICE |
| `signaling-client` | WS-клиент на `java.net.http` |
| `media` | Opus SDP-munger, конвертер аудио Float32 → S16 |
| `media-engine` | webrtc-java: движок, peer-сессии, меш, источники экрана/камеры |
| `app-client` | JavaFX-приложение |

## Запуск

```bash
./gradlew :signaling-server:run     # локальный сервер (порт 8080)
./gradlew :app-client:run           # приложение (в двух терминалах — два участника)
./gradlew build                     # сборка и все тесты
```

В приложении укажите одинаковый код комнаты и разные имена — клиенты установят
P2P-связь и появятся друг у друга со статусом «на связи».

Две копии на одной машине дадут акустическую обратную связь (общий микрофон и
динамики) — для проверки звука используйте две машины или наушники.

## Адрес сервера

В коде адрес не захардкожен (по умолчанию `ws://localhost:8080/signal`). Свой
адрес задаётся одним из источников (по приоритету):

1. `-Dsertas.server=ws://HOST:8080/signal`
2. переменная окружения `SERTAS_SERVER`
3. файл `~/.sertas/server` (одна строка с URL)

Если сервер защищён токеном, он добавляется в URL: `...signal?token=ТОКЕН`.
Поле «Сервер» в окне входа тоже можно отредактировать вручную.

## Статус

Готово и покрыто тестами:

- Сигналинг: протокол, маршрутизация, сервер, клиент, токен-авторизация.
- Реальное P2P-соединение между двумя пирами (ICE/DTLS/SCTP) и полный меш через
  сервер.
- Аудио- и видео-тракты: реальный offer с Opus и `m=video`, музыкальный профиль
  Opus для звука демонстрации.
- Запускаемое приложение: вход в комнату, голосовой звонок, список участников,
  mute, выход.

В планах:

- Захват камеры и экрана, рендер удалённого видео в UI.
- Системный звук демонстрации (нативные хелперы ScreenCaptureKit / WASAPI).
- Переключатели шумоподавления и эхо, пресеты качества, установщики.

## Заметки по реализации

- Нативный артефакт webrtc-java выбирается по платформе сборки
  (`media-engine/build.gradle.kts`).
- Исключение из колбэка WebRTC рушит JVM — все колбэки в `PeerSession`
  обёрнуты и направляют ошибки в `Signals.onError`.
- Нативные объекты освобождаются строго по зависимостям (трек нельзя
  `dispose()`, пока его держит sender).
- В меше offer инициирует пир с меньшим id — обе стороны выбирают одного
  инициатора независимо, без гонки.
- Микрофон идёт через переключаемую обработку (APM); звук демонстрации — отдельным
  треком в обход обработки, стерео Opus.
