# sertas

Кроссплатформенное десктоп-приложение (macOS + Windows; Linux — позже) для
голосовых/видео-звонков с демонстрацией экрана — аналог Discord, сфокусированный
на **качественной демонстрации экрана с высоким FPS и чистым системным звуком**.
До 4 участников в P2P-меше.

- Дизайн: [docs/superpowers/specs/2026-06-24-discord-analog-voice-app-design.md](docs/superpowers/specs/2026-06-24-discord-analog-voice-app-design.md)
- План Фазы 0: [docs/superpowers/plans/2026-06-24-phase-0-signaling-and-foundations.md](docs/superpowers/plans/2026-06-24-phase-0-signaling-and-foundations.md)

## Стек

JDK 21 · JavaFX 21 · [webrtc-java 0.14.0](https://github.com/devopvoid/webrtc-java)
(нативный libwebrtc) · Javalin (сигналинг-сервер) · Gradle (wrapper в комплекте).

## Модули

| Модуль | Ответственность |
|--------|-----------------|
| `protocol` | Сообщения сигналинга (`SignalMessage`) + JSON-кодек |
| `signaling-server` | WebSocket-сервер: комнаты, relay SDP+ICE (логика отделена от транспорта) |
| `signaling-client` | WS-клиент на `java.net.http` |
| `media` | Чистые утилиты: Opus «музыкальный» SDP-munger, конвертер аудио Float32→S16 |
| `media-engine` | webrtc-java: `WebRtcEngine`, `PeerSession`, `MeshCoordinator`, источники экрана/камеры |
| `app-client` | JavaFX-приложение (вход → звонок) |

## Статус

### ✅ Готово и покрыто тестами (30 тестов)
- **Сигналинг** — протокол, маршрутизация, сервер, клиент; сквозной тест «2 клиента через реальный сервер».
- **WebRTC-стек** — нативная библиотека грузится на этом хосте (smoke-тест).
- **Реальное P2P-соединение** — два пира устанавливают связь (ICE→DTLS→SCTP), обмениваются данными по data-channel (loopback-тест).
- **Полный меш** — два `MeshCoordinator` соединяются через реальный сервер (интеграционный тест).
- **Аудио-тракт** — реальный offer содержит Opus; `OpusSdpMunger` применяет музыкальный профиль к настоящему SDP.
- **Видео-тракт** — RGBA→I420→`pushFrame`→трек→offer `m=video` (путь демонстрации экрана).
- **Приложение** — запускаемый JavaFX-клиент: вход в комнату → голосовой звонок → список участников со статусами → mute → выход.

### 🚧 В работе / требует ручной проверки на устройстве
- Реальный захват камеры и экрана (нужны устройство + разрешения TCC) — API готов (`CameraSource`, `ScreenCaptureSource`), UI-кнопки и рендер удалённого видео (`PixelBuffer`) — следующий шаг.
- **Системный звук демонстрации** (Фаза 3) — нативные хелперы ScreenCaptureKit (macOS) / WASAPI loopback (Windows) → `CustomAudioSource`. Главная оставшаяся нативная работа.
- Переключатели шумодав/эхо/AGC (Фаза 4), пресеты качества демо, упаковка jpackage.

## Сборка и запуск

> Gradle не требуется ставить — используется `./gradlew`. Нужен только JDK 21.

```bash
# Все тесты
./gradlew build

# Локальный сигналинг-сервер (порт 8080)
./gradlew :signaling-server:run

# Приложение (в отдельных терминалах — два участника)
./gradlew :app-client:run
```

В приложении: одинаковый **код комнаты**, разные имена — клиенты установят P2P-связь
и появятся друг у друга со статусом «на связи».

### Адрес сигналинг-сервера

В коде адрес НЕ захардкожен. По умолчанию `ws://localhost:8080/signal`. Чтобы
указать свой (например, развёрнутый на VPS — см. [DEPLOY.md](DEPLOY.md)), задайте
любой из источников (в порядке приоритета):

1. `-Dsertas.server=ws://HOST:8080/signal`
2. переменная окружения `SERTAS_SERVER`
3. файл `~/.sertas/server` (одна строка с URL) — удобно для постоянного дефолта

Если сервер защищён токеном (`SERTAS_TOKEN`), добавьте его в URL:
`ws://HOST:8080/signal?token=ВАШ_ТОКЕН`.

Поле «Сервер» в окне входа можно также отредактировать вручную.

> Запуск двух экземпляров на одной машине даст акустическую обратную связь
> (общий микрофон/динамики) — для проверки звука используйте две машины или
> наушники + mute на одном из клиентов.

### Полезные команды

```bash
./gradlew :media-engine:test        # тесты WebRTC-стека (грузят нативную библиотеку)
./gradlew :signaling-server:test    # тесты сигналинга + сквозной
./gradlew test                      # все юнит/интеграционные тесты
```

## Технические заметки

- **Нативный классификатор** webrtc-java выбирается по хосту сборки
  (`macos-aarch64` / `macos-x86_64` / `windows-x86_64` / `linux-x86_64`) — см.
  `media-engine/build.gradle.kts`.
- **Колбэки WebRTC нельзя бросать наружу** — исключение из нативного колбэка
  рушит JVM (`jni::JavaWrappedException`). Все колбэки в `PeerSession` обёрнуты в
  `safe(...)` и направляют ошибки в `Signals.onError`.
- **Освобождение нативных объектов** строго по зависимостям: `dispose()` трека,
  который ещё держит sender, бросает «Native object was not deleted».
- **Анти-glare в меше**: инициатор offer'а — пир с лексикографически меньшим id;
  обе стороны вычисляют одного инициатора независимо, без гонки.
- **Чистый звук**: микрофон идёт через переключаемый APM; звук демонстрации —
  отдельным треком через `CustomAudioSource` в обход обработки + Opus-стерео.
